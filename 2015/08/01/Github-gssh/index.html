生成SSH Key(Generating SSH keys)
SSH Key是一种方法来确定受信任的计算机，而不涉及密码.下面的步骤将引导您完成生成SSH Key并添加public Key到你的Github帐号.
我们建议您定期检查您的SSH Key列表，并撤销尚未在一段时间使用的SSH Key.

提示：如果您在Windows上安装了github for Windows，你可以用它来克隆库，而不是处理SSH Key.它还配备了Git Bash 工具，这是在Windows上运行的git命令的首选方式

第1步：检查SSH Key(Step 1: Check for SSH keys)

首先，我们需要检查您的计算机上现有的SSH Key.打开的Git Bash输入：

检查清单，看看您是否已经有了一个SSH Key的目录.默认情况下，public Key文件名是下列之一：

  • id_dsa.pub
  • id_ecdsa.pub
  • id_ed25519.pub
  • id_rsa.pub

如果您看到列出现有的public 和 private Key（例如id_rsa.pub和id_rsa），你想使用连接到GitHub上，你可以跳过步骤2，直接进入步骤3.

提示：如果您收到 ~/.ssh doesn’t exist, don’t worry! 不要担心！我们将在第2步创建它.

添加复制的KEY到GitHub上：
1.在任何页面的右上角，单击您的个人资料照片，然后单击Settings。
2.在用户Settings侧边栏，单击SSH Keys。
3.点击Add SSH Key。
4.在标题字段中，为新的Key添加描述性标签。例如，如果您使用的是Mac的人，你可能把这种Key写上“Personal MacBook Air”。
5.粘贴Key到你的“Key”区域中。
6.单击 Add Key
7.确认输入您的GitHub密码。
